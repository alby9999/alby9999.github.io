Setup: Project Structure & Installation

Before writing any code, set up your environment. Open a terminal in a new folder called lab9/.

Step 0 — Create & Activate a Virtual Environment

A virtual environment keeps your project's dependencies isolated from your system Python. Always create one before installing packages.

macOS / Linux:

python3 -m venv venv
source venv/bin/activate

Windows (Command Prompt):

python -m venv venv
venv\Scripts\activate

Windows (PowerShell):

python -m venv venv
venv\Scripts\Activate.ps1

Once activated, your terminal prompt will show (venv) at the start. To deactivate later, run deactivate.

Part 1: Your First Flask App & Routes

Flask maps URL paths to Python functions called view functions using the @app.route() decorator. Open app.py.

Task 1.1 — Create the Flask App Instance

In app.py, complete the TODO to create a Flask application instance. This object is the core of your web app.

  1. Import Flask, render_template, and abort from the flask module.
  2. Create a Flask app instance assigned to a variable named app. Pass __name__ as the argument.
Hint: app = Flask(__name__) — __name__ tells Flask where to find resources like the templates/ folder.

Task 1.2 — Define the Home Route

A route connects a URL to a Python function. Complete the home route in app.py:

  1. Add the @app.route('/') decorator above the index() function.
  2. For now, make the function return a plain string: "Hello, Flask!".
  3. Run the app: python app.py and visit http://127.0.0.1:5000/ in your browser to confirm it works.

Task 1.3 — Define the About Route

Add a second route for the About page:

Part 2: Rendering HTML Templates

Returning raw strings is impractical. Flask uses Jinja2 to render HTML files from the templates/ folder. The syntax {{ variable }} injects a Python value into HTML.

Task 2.1 — Render the Home Template

Update the index() function in app.py:

  1. Replace the plain string return with return render_template('index.html', ...).
  2. Pass the posts list (already defined above the route) as a keyword argument: posts=posts.
  3. Also pass title="Home".

Task 2.2 — Display the Post List in index.html

Open templates/index.html and complete the Jinja2 template:

  1. Use {{ title }} to display the page title in the <h1> tag.
  2. Use a Jinja2 for loop to iterate over posts: 
    {% for post in posts %}
  ...
{% endfor %}
  3. Inside the loop, display each post's id, title, and author fields using {{ }} syntax.
  4. Wrap each post in an <article> tag.
Hint: Access a dictionary key in a template with dot notation: {{ post.title }} or bracket notation: {{ post['title'] }}.

Task 2.3 — Render the About Template

Update the about() function in app.py:

  1. Use render_template('about.html', title="About").
  2. In templates/about.html, use {{ title }} in the heading and write a short description of the blog (any text you like).

Part 3: Template Inheritance

Repeating the same <head>, nav bar, and footer in every template is messy. Flask/Jinja2 solves this with template inheritance: a base template defines the shared layout, and child templates fill in the specific content.

Key syntax:

TagWhere UsedPurpose
{% block name %}{% endblock %}base.htmlDeclares a named placeholder
{% extends "base.html" %}child templatesInherit the base layout
{% block name %}...{% endblock %}child templatesOverride the placeholder with content

Task 3.1 — Complete the Base Template

Open templates/base.html and complete the TODOs:

  1. In the <title> tag, add a block called title with a default value of Student Blog.
  2. In the <main> tag, add a block called content (leave it empty — child templates will fill it).
  3. The nav bar links to / (Home) and /about (About) are already provided — review them.

Task 3.2 — Extend the Base in index.html

Update templates/index.html to use the base layout:

  1. Add {% extends "base.html" %} as the very first line.
  2. Wrap your existing content inside {% block content %} ... {% endblock %}.
  3. Add {% block title %}Home — Student Blog{% endblock %} to set the browser tab title.
  4. Reload http://127.0.0.1:5000/ — the nav bar from base.html should now appear.

Task 3.3 — Extend the Base in about.html

Apply the same inheritance pattern to templates/about.html:

  1. Add the extends tag at the top.
  2. Wrap your about content in a content block.
  3. Set a suitable browser title using the title block.

Part 4: Dynamic Routes & 404 Handling

Dynamic routes let a single route handle many URLs by capturing a variable segment from the URL path. For example, /post/3 and /post/7 both match /post/<int:post_id>, passing 3 or 7 as the argument.

Task 4.1 — Define the Dynamic Post Route

In app.py, complete the post(post_id) view function:

  1. Add the route decorator: @app.route('/post/<int:post_id>').
  2. Search the posts list for the post whose 'id' matches post_id.
  3. If no matching post is found, call abort(404) to return a 404 error.
  4. If found, call render_template('post.html', post=post, title=post['title']).
Hint: Use a Python next() with a generator expression to find the post: 
post = next((p for p in posts if p['id'] == post_id), None)
If post is None, call abort(404).

Task 4.2 — Build the Post Template

Open templates/post.html and complete the TODOs:

  1. Extend base.html and set the title block to display the post title.
  2. In the content block, display:
    • The post title in an <h1>.
    • The post author and date in a <p> tag.
    • The post body in a <p> tag.
    • A <a href="/">Back to Home</a> link.

Task 4.3 — Link Posts from the Home Page

Update templates/index.html so each post title is a clickable link:

  1. Wrap the post title in an anchor tag that links to /post/{{ post.id }}.
  2. Use Flask's url_for function for best practice: {{ url_for('post', post_id=post.id) }}.
  3. Add url_for to your import at the top of app.py.
  4. Click a post title on the home page and confirm the post detail page loads correctly.
Why url_for? Hard-coded links like /post/1 break if you change your route. url_for('post', post_id=1) always generates the correct URL from the function name, even if the route changes.

Task 4.4 — Custom 404 Error Page

Add a friendly error page when a post ID does not exist:

  1. In app.py, register a 404 error handler using the @app.errorhandler(404) decorator above a function called not_found(error).
  2. Inside the function, return render_template('404.html'), 404.
  3. Create templates/404.html that extends base.html and displays a friendly "Page Not Found" message with a link back to home.
  4. Test by visiting http://127.0.0.1:5000/post/999 — you should see your custom page.

Bonus

Extend the application with atleast one (or more) of the following:

  • Tag filter: Add a tags list to each post dict and create a route /tag/<tag_name> that shows all posts with that tag.
  • Author page: Add a route /author/<string:name> that lists all posts by that author.
